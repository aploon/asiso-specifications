  1. Introduction

    1. Purpose

      The purpose of this Software Requirement Specification (SRS) is to outline the functional and non-functional requirements for the Asiso - Quality Management System (QMS) Application developed by Takka Technologies Inc. The SRS serves as a reference document for the development team and stakeholders to ensure a common understanding of the application's features, functionalities, and constraints.


    2. Scope

      The Asiso application is a QMS solution designed to assist organizations in managing and improving their quality processes in accordance with the AS9100D standard. The application integrates seamlessly with various Microsoft applications, including MS Teams, MS Word, MS Excel, and SharePoint. It provides features such as document management, nonconformance management, risk management, training and competency management, and reporting and analytics.

      The scope of this SRS includes the detailed specifications for the MVP (Minimum Viable Product) version of the Asiso application. The MVP includes essential features and functionalities that meet the basic requirements of a QMS. Additional features and enhancements may be considered in future versions.


    3. Definitions, Acronyms, and Abbreviations

      • QMS: Quality Management System

      • SRS: Software Requirement Specification

      • AS9100D: Aerospace Quality Management System standard

      • MVP: Minimum Viable Product

      • SAAS: Software as a Service

      • RBAC: Role-Based Access Control

      • MFA: Multi-Factor Authentication

      • OAuth: Open Authorization

      • API: Application Programming Interface

      • AES: Advanced Encryption Standard


    4. References

      • AS9100D: Quality Management Systems - Requirements for Aviation, Space, and Defense Organizations


    5. Overview

      The Asiso application is developed using Microsoft technologies and takes advantage of the Microsoft Action Pack benefits. It is offered as a Software as a Service (SAAS) solution, enabling organizations to subscribe and pay per user per month.

      The application aims to streamline quality management processes, enhance collaboration among team members, ensure compliance with the AS9100D standard, and facilitate data-driven decision-making. It integrates seamlessly with popular Microsoft applications to provide a familiar and cohesive user experience.

      The following sections of this SRS provide detailed specifications for the various features and functionalities of the Asiso application, including user registration and login, document management, nonconformance management, reporting and analytics, integration with Microsoft applications, SAAS mode, user management, data security, and traceability to AS9100D standard clauses.


  2. General Description

    1. Product Perspective

      The Asiso application is a standalone QMS solution that operates independently of other software systems. However, it is designed to integrate seamlessly with various Microsoft applications, including MS Teams, MS Word, MS Excel, and SharePoint. The integration allows users to leverage existing tools and workflows while accessing and managing quality-related data within the Asiso application.

      The Asiso application interacts with external systems through APIs and connectors to enable data exchange and synchronization. It utilizes Microsoft technologies and services to ensure a secure, reliable, and scalable solution.


    2. Product Features

      The Asiso application offers the following key features:

      • User Registration and Login: Users can create accounts and authenticate themselves securely using multi-factor authentication (MFA). User roles and permissions are defined to control access to different application functionalities.

      • Document Management: The application provides comprehensive document management capabilities. Users can create, edit, and store quality-related documents using predefined templates or custom formats. Document approval workflows ensure proper review and authorization processes.

      • Nonconformance Management: Users can report nonconformances, track their resolution status, and assign corrective actions. Workflows facilitate the investigation, root cause analysis, and implementation of corrective measures.

      • Reporting and Analytics: Pre-built reports and customizable dashboards enable users to monitor key quality metrics, track compliance, and identify trends. Data analysis tools provide insights for decision-making and continuous improvement.

      • Integration with Microsoft Applications: Seamless integration with MS Teams, MS Word, MS Excel, and SharePoint allows users to collaborate on quality-related tasks, access documents, and share information easily.

      • SAAS Mode: The Asiso application follows a SAAS pricing model, allowing organizations to subscribe and pay per user per month. It offers scalability to accommodate varying user needs and provides data backup and recovery mechanisms.



      • User Management: User registration, role assignment, and password management functionalities are provided. The application supports role-based access control (RBAC) to ensure appropriate access privileges for different user types.

      • Data Security: The application incorporates robust security measures, including data encryption using the Advanced Encryption Standard (AES). User authentication and access control mechanisms ensure data confidentiality and integrity.


    3. User Classes and Characteristics

      The Asiso application caters to different user classes, each with specific characteristics and roles within the QMS:

      • Administrators: Responsible for system configuration, user management, and overall QMS administration.

      • Quality Managers: Oversee quality processes, monitor performance, and ensure compliance with standards.

      • Document Controllers: Manage the creation, approval, and version control of quality-related documents.

      • Nonconformance Managers: Handle nonconformance reports, assign corrective actions, and track their resolution.

      • Risk Managers: Identify and assess risks, define mitigation plans, and monitor their effectiveness.

      • Training Coordinators: Identify training needs, manage training programs, and track employee competencies.

      • Employees: Interact with the system, report nonconformances, access documents, and complete assigned tasks.


    4. Operating Environment

      The Asiso application is designed to run in a web browser environment. It is compatible with modern browsers such as Google Chrome, Mozilla Firefox, Microsoft Edge, and Safari. The application is developed using Microsoft technologies, making it suitable for deployment on Windows-based servers.

      The application requires an internet connection for accessing the SAAS infrastructure and integrating with Microsoft applications. It is designed to handle concurrent user sessions and maintain high performance and responsiveness.


    5. Design and Implementation Constraints

      The design and implementation of the Asiso application should adhere to the following constraints:

      • Microsoft Technologies: The application must be developed using Microsoft technologies, frameworks, and APIs to ensure seamless integration with Microsoft applications and leverage the Microsoft Action Pack benefits.

      • Compliance with AS9100D: The application must comply with the AS9100Dstandard requirements to support organizations in achieving and maintaining certification. It should align with the clauses and principles outlined in the AS9100D standard.

      • Security and Data Privacy: The application must prioritize data security and implement appropriate measures to protect sensitive information. It should comply with relevant data privacy regulations, such as GDPR (General Data Protection Regulation) or applicable industry-specific standards.

      • Scalability: The application should be designed to handle varying user loads and accommodate the growth of the organization. It should support scalability by efficiently utilizing server resources and adopting scalable infrastructure components.

      • Usability and User Experience: The application should provide a user-friendly interface with intuitive navigation and clear workflows. It should consider accessibility guidelines to ensure inclusivity for users with disabilities.

      • Integration with Microsoft Applications: The integration with MS Teams, MS Word, MS Excel, and SharePoint should be seamless, allowing users to collaborate and access quality-related data without disruptions.


    6. User Documentation

      The Asiso application requires comprehensive user documentation to assist users in understanding its features, functionalities, and workflows. The documentation should include:

      • User Manuals: Detailed guides explaining how to use each feature and perform common tasks within the application.

      • Administrator Guides: Instructions for system configuration, user management, and customization options.

      • API Documentation: If applicable, documentation for developers outlining the APIs and connectors available for integration purposes.


      The documentation should be easily accessible from within the application and available in both online and downloadable formats.


    7. Assumptions and Dependencies

      The development and successful implementation of the Asiso application are based on the following assumptions and dependencies:

      • Internet Connectivity: Users must have reliable internet connectivity to access and utilize the application's features.

      • Microsoft Application Integration: The successful integration of the Asiso application with Microsoft applications depends on the availability and compatibility of their APIs and connectors.

      • Compliance Knowledge: Users and administrators of the application are assumed to have a basic understanding of quality management principles and the AS9100D standard.

      • Server Infrastructure: The organization must have suitable server infrastructure or choose to deploy the application in a cloud environment that meets the system requirements and provides adequate performance and data security.

      • Licensing and Subscription: Users must acquire appropriate licensing or subscribe to the Asiso SAAS offering to access the application and benefit from its features and support.

      • Training and Support: Organizations are responsible for providing adequate training and support to users during the onboarding and ongoing usage of the Asiso application.

      • Regulatory Compliance: Organizations using the Asiso application must ensure compliance with relevant industry regulations and standards in addition to the AS9100D standard.


  3. Specific Requirements

    1. User Registration and Login

      1. Requirement: Users shall be able to create accounts by providing necessary information such as name, email address, and password.

      2. Requirement: User registration shall support multi-factor authentication (MFA) for enhanced security.

      3. Requirement: The application shall provide a login page where users can securely authenticate using their registered email address and password.

      4. Requirement: The application shall support password reset functionality for users who have forgotten their passwords.


    2. Document Management

      1. Requirement: Users with appropriate permissions shall be able to create new documents within the application.

      2. Requirement: The application shall support document templates for common quality-related documents, such as procedures, work instructions, and forms.

      3. Requirement: Users shall be able to upload, edit, and version control documents within the application.

      4. Requirement: Document approval workflows shall be implemented to ensure proper review and authorization processes.

      5. Requirement: The application shall provide a search functionality to enable users to locate documents based on keywords, metadata, or tags.


    3. Nonconformance Management

      1. Requirement: Users shall be able to report nonconformance incidents, providing relevant details such as description, category, and severity.

      2. Requirement: Nonconformance incidents shall follow a workflow that includes investigation, root cause analysis, and corrective action implementation.

      3. Requirement: The application shall provide the ability to assign and track corrective actions associated with nonconformance incidents.

      4. Requirement: Users with appropriate permissions shall be able to monitor the status and progress of nonconformance incidents and associated corrective actions.


    4. Reporting

      1. Requirement: The application shall provide pre-built reports that display key quality metrics, such as nonconformance trends, corrective action effectiveness, and process performance.

      2. Requirement: Users shall be able to generate customizable reports based on selected criteria and parameters.

      3. Requirement: The application shall support exporting reports to common formats, such as PDF or Excel, for further analysis or distribution.


    5. Integration with Microsoft Applications

      1. Requirement: The application shall seamlessly integrate with MS Teams, allowing users to access quality-related documents, receive notifications, and collaborate on tasks within Teams channels.

      2. Requirement: Users shall be able to create or edit documents directly from MS Word, with changes automatically synced to the Asiso application.

      3. Requirement: Users shall be able to import or export data between the Asiso application and MS Excel for data analysis or bulk updates.

      4. Requirement: The application shall provide integration with SharePoint, allowing users to store and manage quality-related documents in SharePoint libraries.


    6. SAAS Mode

      1. Requirement: The application shall be provided as a SAAS solution, enabling organizations to subscribe and pay per user per month.

      2. Requirement: The application's pricing model shall be flexible, allowing organizations to scale the number of users up or down as needed.

      3. Requirement: The application shall implement regular data backups and provide a data recovery mechanism in case of data loss or system failure.


    7. User Management

      1. Requirement: Administrators shall have the ability to manage user registrations, including adding, editing, and deactivating user accounts.

      2. Requirement: The application shall support role-based access control (RBAC), allowing administrators to assign different roles and permissions to users based on their responsibilities and organizational hierarchy.


      3. Requirement: User roles shall determine the level of access and functionality available to each user within the application.

      4. Requirement: The application shall provide password management functionality, allowing users to change their passwords and enforce password complexity rules.


    8. Data Security

      1. Requirement: The application shall implement data encryption using the Advanced Encryption Standard (AES) to ensure the confidentiality and integrity of sensitive data.

      2. Requirement: User authentication shall be performed securely, following industry best practices and using encryption protocols.

      3. Requirement: The application shall have mechanisms in place to prevent unauthorized access and protect against common security threats, such as cross-site scripting (XSS) and SQL injection.

      4. Requirement: The application shall comply with relevant data privacy regulations, such as GDPR, to safeguard user data and ensure compliance with privacy requirements.

    9. Traceability to AS9100D Standard Clauses

      1. Requirement: The application shall provide traceability to the AS9100D standard clauses, enabling users to map their quality management activities and processes to the corresponding clauses.

      2. Requirement: Users shall be able to generate reports or visualizations that demonstrate compliance with specific AS9100D clauses and requirements.

      3. Requirement: The application shall support updates to the AS9100D standard and provide mechanisms to incorporate changes into the system, ensuring continued compliance with the latest version of the standard.


  4. System Requirements

    1. Hardware Requirements

      1. Requirement: The Asiso application shall be compatible with commonly used desktop and laptop computers.

      2. Requirement: The minimum system requirements for running the application shall include a modern processor (e.g., Intel Core i5 or equivalent), 8 GB of RAM, and a minimum of 100 GB of available hard disk space.

      3. Requirement: The application shall be compatible with major operating systems, including Windows, macOS, and Linux.


    2. Software Requirements

      1. Requirement: The application shall be accessible via standard web browsers, including Google Chrome, Mozilla Firefox, Microsoft Edge, and Safari.

      2. Requirement: The application shall be developed using Microsoft technologies, such as .NET Framework or .NET Core.

      3. Requirement: The application shall utilize Microsoft Azure or other cloud hosting platforms for SAAS deployment.

      4. Requirement: The application shall integrate with Microsoft applications, including MS Teams, MS Word, MS Excel, and SharePoint, using their respective APIs and connectors.

      5. Requirement: The application shall utilize a relational database management system (RDBMS) compatible with Microsoft technologies, such as Microsoft SQL Server or Azure SQL Database.


    3. Performance Requirements

      1. Requirement: The application shall provide responsive performance, ensuring smooth user interactions and minimal latency.

      2. Requirement: The application shall be designed to handle concurrent user sessions without degradation in performance.

      3. Requirement: The application shall have mechanisms in place to handle anticipated user loads, scaling resources as necessary to maintain optimal performance.

      4. Requirement: The application shall support efficient search functionality, providing fast and accurate results even when handling large document repositories.


    4. Availability and Reliability

      1. Requirement: The application shall have a minimum uptime of 99.9%, excluding scheduled maintenance windows.

      2. Requirement: The application shall implement fault tolerance and disaster recovery mechanisms to ensure data availability and minimize downtime.

      3. Requirement: The application shall perform regular data backups to protect against data loss and provide options for data restoration.


    5. Security Requirements

      1. Requirement: The application shall employ secure data transmission protocols, such as HTTPS, to protect user data during transit.

      2. Requirement: The application shall enforce strong password policies, including minimum length, complexity, and expiration requirements.

      3. Requirement: The application shall implement role-based access control (RBAC), ensuring that users can only access and perform actions within their authorized scope.

      4. Requirement: The application shall have mechanisms in place to detect and prevent common security threats, such as cross-site scripting (XSS) and SQL injection.

      5. Requirement: The application shall provide audit logs to track user activities, system changes, and data access for security and compliance purposes.


    6. Compatibility Requirements

      1. Requirement: The application shall be compatible with the latest versions of Microsoft Office applications, including MS Word and MS Excel, to ensure seamless document editing and integration.

      2. Requirement: The application shall support integration with different versions of MS Teams and SharePoint, ensuring compatibility with varying organizational setups.


  5. Design and Architecture

    1. System Architecture

      1. Requirement: The Asiso application shall follow a multi-tier architecture, separating the presentation layer, business logic layer, and data layer.

      2. Requirement: The application shall utilize a scalable and distributed architecture to accommodate increasing user loads and ensure optimal performance.

      3. Requirement: The application shall employ RESTful APIs for communication between different components and modules.

      4. Requirement: The system architecture shall allow for future extensibility and integration with additional modules or third-party systems.


    2. User Interface Design

      1. Requirement: The application's user interface shall follow modern design principles, including simplicity, consistency, and responsiveness.

      2. Requirement: The user interface shall be intuitive and user-friendly, minimizing the learning curve for new users.

      3. Requirement: The application shall provide a responsive design, ensuring optimal user experience across different devices and screen sizes.

      4. Requirement: The user interface shall support customization options, allowing users to personalize their view and preferences.


    3. Data Management

      1. Requirement: The application shall utilize a robust database management system to store and manage data, such as Microsoft SQL Server or Azure SQL Database.

      2. Requirement: The database design shall be optimized for performance and scalability, considering factors such as indexing, query optimization, and data partitioning.

      3. Requirement: The application shall implement data encryption at rest to protect sensitive information stored in the database.

      4. Requirement: The database shall maintain proper data relationships and enforce referential integrity to ensure data consistency and accuracy.


    4. Integration with Microsoft Applications

      1. Requirement: The application shall utilize Microsoft APIs and connectors for seamless integration with MS Teams, MS Word, MS Excel, and SharePoint.

      2. Requirement: The integration with MS Teams shall allow users to access quality-related documents, receive notifications, and perform tasks directly within Teams channels.

      3. Requirement: The integration with MS Word shall enable users to create or edit documents, with changes automatically synchronized between the Asiso application and Word.

      4. Requirement: The integration with MS Excel shall facilitate data import and export between the Asiso application and Excel for analysis or bulk updates.

      5. Requirement: The integration with SharePoint shall enable users to store and manage quality-related documents in SharePoint libraries, ensuring seamless document collaboration and version control.


    5. API and Integration Framework

      1. Requirement: The application shall provide a well-documented API framework, allowing third-party systems to integrate with the Asiso application.

      2. Requirement: The API framework shall support authentication and authorization mechanisms to ensure secure access to the application's data and functionalities.

      3. Requirement: The API framework shall support standard data interchange formats, such as JSON or XML, for seamless data integration with external systems.

      4. Requirement: The application shall provide connectors or plugins for popular integration platforms, such as Microsoft Power Automate or Azure Logic Apps, to facilitate data exchange and workflow automation.


    6. Customization and Extensibility

      1. Requirement: The application shall allow for customization and configuration of certain features, such as document templates, workflows, and user roles.

      2. Requirement: The customization options shall be user-friendly and accessible to administrators or power users without requiring extensive technical knowledge.

      3. Requirement: The application shall provide extensibility options, allowing for the addition of new modules, functionalities, or integrations based on specific business requirements.

      4. Requirement: The extensibility mechanisms shall follow industry best practices, such as modular design, well-defined APIs, and separation of concerns, to ensure maintainability and scalability.


    7. Performance Optimization

      1. Requirement: The application shall implement caching mechanisms to enhance performance and reduce database load.

      2. Requirement: The application shall employ techniques such as lazy loading and pagination to optimize the retrieval and display of large datasets.

      3. Requirement: The application shall utilize asynchronous processing and background tasks for computationally intensive operations, ensuring a responsive user experience.

      4. Requirement: The system architecture shall incorporate load balancing and horizontal scaling techniques to distribute user requests and maximize resource utilization.


    8. Error Handling and Logging

      1. Requirement: The application shall have robust error handling mechanisms to handle exceptions and prevent application crashes or data corruption.

      2. Requirement: The application shall log system events, errors, and user activities to facilitate troubleshooting, auditing, and system monitoring.

      3. Requirement: The log files shall adhere to standardized formats and include relevant information, such as timestamps, severity levels, and user context.

    9. Localization and Internationalization

      1. Requirement: The application shall support multiple languages and provide localization options for different regions and locales.

      2. Requirement: The user interface shall be designed to accommodate different language translations and support right-to-left (RTL) languages.

      3. Requirement: The application shall provide mechanisms for language selection and dynamic translation of user interface elements and content.
